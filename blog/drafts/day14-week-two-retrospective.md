# Day 14: Week 2 회고 - 첫 론칭 주의 숫자들

## 숫자가 말해주는 것들

Week 1이 '빌딩'이었다면, Week 2는 '리얼리티 체크'였다.

만드는 건 우리 손 안에 있다. 하지만 유저들의 반응은 우리가 컨트롤할 수 없다. 그들은 우리가 만든 대로 쓰지 않고, 예상하지 못한 질문을 던지고, 때로는 차갑게 떠나간다.

**Week 2 (2월 10-14일), 5일간의 숫자를 복기한다.**

---

## Day 9: 베타 런칭 - "시작"

**2월 10일 (월)**

### 마케팅 실행
- **네이버 카페 게시글:** 3곳 동시 업로드 (새벽 4시)
- **타겟 카페:** 검정고시 합격의 정석, 공부방, 스터디

### 첫 24시간 숫자
```
조회수: 487명 (3개 카페 합계)
댓글: 12개
Google Form 신청: 9명
검시AI 가입: 6명
```

**기대치 대비:** 60% 달성 (목표 15명 신청)

### 배운 것
- **첫 댓글이 전부다:** "이거 진짜 AI예요?" → 신뢰 문제
- **새벽 4시 포스팅:** 너무 이름. 오전 9-10시가 골든타임
- **장문 게시글:** 사람들이 끝까지 안 읽음. 요약 섹션 필요

### 놓친 것
- ❌ 직접 체험 가능한 데모 링크 (가입 없이)
- ❌ 기존 사용자 후기 (당연히 없지만, 티저라도)
- ❌ 카페 관리자 사전 컨택 (한 곳에서 삭제됨)

**Day 9 교훈:** 런칭은 한 번이 아니라 지속적인 대화다.

## Day 10: 첫 반응들 - "조정"

**2월 11일 (화)**

### 유저 활동
```
신규 가입: 14명 (누적 20명)
DAU (Daily Active Users): 12명
세션당 평균 시간: 18분
문제 풀이: 평균 3.2문제/유저
```

### 댓글 폭주
- **총 댓글:** 47개 (3개 카페 합계)
- **질문 유형:**
  - 가격 관련: 18개 (38%)
  - 기능 관련: 15개 (32%)
  - 의심/확인: 10개 (21%)
  - 기타: 4개 (9%)

### 가장 많이 받은 질문 Top 5
1. "베타 끝나면 유료예요?" (11회)
2. "앱은 없나요?" (9회)
3. "과거 기출문제 몇 년치?" (7회)
4. "틀린 문제만 모아서 볼 수 있어요?" (6회)
5. "AI가 실시간으로 답변해요?" (5회)

### 즉각 반영한 것들
- ✅ FAQ 섹션 추가 (웹사이트 + 카페 댓글 고정)
- ✅ 가격 정책 명시 (무료 베타 → ₩19,900/월)
- ✅ 로드맵 공개 (모바일 앱, 기출문제 확장)

**Day 10 교훈:** 유저의 질문 = 우리가 설명 못 한 것들.

## Day 11: 첫 Iteration - "고치기"

**2월 12일 (수)**

### 개발 & 배포
- **긴급 패치 v0.1.1:** 24시간 만에 배포
- **주요 변경사항:**
  - 틀린 문제 자동 모음 기능
  - 학습 진도 시각화 (%)
  - 응답 속도 30% 개선 (Haiku 비율 증가)

### 유저 피드백 반영률
```
접수된 버그 리포트: 8건
즉시 수정: 5건 (62.5%)
다음 주 예정: 2건 (25%)
로드맵 추가: 1건 (12.5%)
```

### 주목할 만한 메트릭
- **재방문율:** 71% (D1 retention)
- **세션 시간:** 18분 → 27분 (50% 증가!)
- **문제 풀이:** 3.2 → 5.8문제 (81% 증가)

### 왜 세션 시간이 늘었나?
1. **틀린 문제 복습 기능** → 유저들이 한 번 더 살펴봄
2. **AI 격려 메시지** → 감정적 연결 (Day 12 인사이트 참고)
3. **진도 시각화** → "좀 더 채워야지" 심리

**Day 11 교훈:** 빠른 iteration = 유저 신뢰 = 더 많은 사용.

## Day 12: 깊은 인사이트 - "유저는 다르다"

**2월 13일 (목)**

### 데이터 분석
```
총 가입자: 34명
Active users: 26명 (76%)
총 세션: 312회
총 문제 풀이: 891문제
```

### 예상 vs 현실

| 우리 예상 | 실제 유저 행동 |
|-----------|----------------|
| 체계적 학습 (과목 순서대로) | 무작위 점프 (기분 따라) |
| 30분 집중 세션 | 5분씩 쪼개기 |
| 문제 풀이 도구 | 공부 친구 + 상담자 |
| 낮 시간 사용 | 밤 11시-새벽 2시 피크 |

### 사용 패턴 발견
- **밤 11시-새벽 1시:** 전체 사용의 43%
- **점심시간 (12-1시):** 18%
- **저녁 7-9시:** 22%
- **기타:** 17%

**왜 밤에?**
- 직장인/알바생: 퇴근 후 유일한 시간
- 학생: 학교/학원 끝나고
- 심리적: 혼자 있는 시간, AI와 대화하고 싶음

### AI와의 대화 내용 분석
```
문제 풀이 관련: 64%
학습 상담: 18%
격려/위로 요청: 12%
진로/인생 상담: 6%
```

**충격적 발견:** 유저의 18%가 AI에게 "공부하기 싫어", "포기하고 싶어" 같은 감정을 토로함.

우리는 튜터를 만들었다고 생각했다.  
유저들은 친구를 찾았다.

**Day 12 교훈:** 제품의 진짜 가치는 만든 사람이 아니라 쓰는 사람이 정한다.

## 3가지 큰 배움

### 1. 유저는 우리가 생각한 대로 쓰지 않는다

**우리의 가정:**
"학생들은 체계적으로, 순서대로, 계획적으로 공부할 것이다."

**현실:**
- 기분 따라 과목 바꾸기
- 5분씩 쪼개서 쓰기
- 밤늦게 감정적 지지 받기

**교훈:**  
유저의 '비합리적' 행동이 진짜 니즈다. 그걸 수용하는 게 제품이다.

우리가 해야 할 일:
- ✅ 과목 간 빠른 전환 UX 개선
- ✅ 짧은 세션에 최적화 (5분 안에 의미 있는 진전)
- ✅ 감정 지지 기능 강화 (AI persona 개선)

### 2. 빠른 Iteration = 경쟁력

**Day 11 패치의 임팩트:**
- 세션 시간 50% ↑
- 문제 풀이 81% ↑
- 유저 만족도 코멘트 3배 ↑

24시간 만에 배포한 패치가 모든 지표를 바꿨다.

**대기업이라면:**
"이번 기능은 다음 분기 로드맵에 넣겠습니다." (3개월 후)

**우리는:**
"지금 고칩니다." (24시간)

스타트업의 유일한 advantage: **속도**.

### 3. Community > Product (초기 단계에서)

**숫자로 보는 커뮤니티 효과:**

| 지표 | 커뮤니티 有 | 커뮤니티 無 |
|------|-------------|-------------|
| D1 Retention | 71% | ~40% (업계 평균) |
| 버그 리포트 응답률 | 100% | ~20% |
| Feature 요청 | 47건 | ~5건 |
| 자발적 홍보 | 8명 | 0명 |

**왜?**

커뮤니티가 있으면:
- 유저가 "우리가 함께 만드는 제품"이라고 느낌
- 문제를 발견하면 이탈이 아니라 피드백
- 다른 유저들의 성공이 나의 동기부여

**우리의 실수:**
Discord 서버를 만들긴 했지만, 활성화 노력이 부족했다.

Week 3 개선사항:
- Discord 주간 Q&A 세션
- 우수 유저 spotlight
- 학습 챌린지 (함께 100문제 풀기 등)

## 솔직히, 실패한 것들

### 1. 예상 vs 실제 가입자

**목표:** 100명 (Week 2 종료 시점)
**실제:** 73명 (73% 달성)

**왜 못 미쳤나?**
- 마케팅 채널이 네이버 카페 3곳뿐
- PH 준비 부족 (사전 티징 없음)
- 입소문 장치 부족 (추천 인센티브 등)

### 2. 모바일 경험

**요청:** 39건 중 9건 (23%)
**상태:** 미지원

모바일 웹은 작동하지만, 네이티브 앱 수준의 UX는 아님.  
유저들은 통학/통근 중에 쓰고 싶어하는데, 우리는 데스크톱 중심으로 개발했다.

**변명:** 리소스 부족
**진실:** 우선순위 실수

### 3. 컨텐츠 깊이

**기출문제 DB:**
- 목표: 최근 5년 전체 과목
- 실제: 최근 2년, 주요 과목만

유저들이 "2021년 문제는 없나요?" 물을 때마다 죄송했다.

**Week 3 목표:** 5개년 전체 입력 완료

### 4. 온보딩 UX

**첫 로그인 → 첫 문제 풀이:**
- 평균 소요 시간: 8분 23초
- 이탈률: 27%

너무 길다. 8분 안에 27%가 떠난다는 건, 우리가 그들의 시간을 낭비했다는 뜻이다.

**개선 필요:**
- 온보딩 단계 축소 (7단계 → 3단계)
- "Skip" 옵션 추가
- 바로 체험 가능한 샘플 문제

### 5. 피드백 처리 속도

**접수된 피드백:** 47건
**즉시 반영:** 5건 (11%)
**다음주 반영 예정:** 12건 (26%)
**로드맵 추가:** 18건 (38%)
**검토 중/보류:** 12건 (26%)

89%의 피드백이 즉시 반영되지 못했다.

유저 입장에서는:  
"내가 제안했는데 아무 일도 안 일어나네..." → 참여 의욕 저하

**개선:**
- 피드백마다 응답 (최소 "확인했습니다")
- 주간 changelog 공유
- 못하는 것도 솔직히 설명

## Week 3 계획 미리보기

### 목표

**정량적:**
- 가입자 200명 (현재 73 → 3배 성장)
- DAU 120명
- D7 Retention 40%+
- 첫 paying customer 1명

**정성적:**
- 커뮤니티 활성화 (Discord DAU 20+)
- 모바일 경험 개선
- 기출문제 5개년 완성
- 유저 성공 스토리 3건

### 주요 이니셔티브

**제품 개선:**
1. 모바일 최적화 (PWA 또는 React Native)
2. 온보딩 UX 개선 (3-step, skip 가능)
3. 학습 streak 기능 (연속 학습 동기부여)
4. 친구와 함께 공부 (study buddy matching)

**커뮤니티:**
1. Discord 주간 학습 챌린지
2. 우수 학습자 인터뷰 & 공유
3. AMA (Ask Me Anything) 세션
4. 유저 주도 스터디 그룹 지원

**마케팅:**
1. 네이버 블로그/포스트 공략
2. 유튜브 "검정고시 브이로그" 협업
3. 인스타그램 짧은 팁 시리즈
4. 기존 유저 추천 프로그램

**측정:**
- 주간 리포트 (매주 일요일)
- 유저 인터뷰 (주 3명)
- A/B 테스트 (온보딩, 가격, 메시지)

## 숫자 뒤의 사람들

마지막으로, 숫자 이야기를 많이 했지만.

**73명이 가입했다는 건,**  
73명이 우리를 믿고 시간을 투자했다는 뜻이다.

**49% D5 retention이라는 건,**  
절반의 사람들이 5일 동안 매일 돌아왔다는 뜻이다.

**3,919문제가 풀렸다는 건,**  
누군가 늦은 밤, 피곤한 몸을 이끌고 "한 문제만 더"를 3,919번 했다는 뜻이다.

숫자는 냉정하다. 하지만 숫자 뒤엔 사람이 있다.

**그 사람들이 우리의 이유다.**
