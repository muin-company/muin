# Blog Post Publishing Checklist

## Pre-Publication Checks
- [ ] Verify date in frontmatter (2026-02-08)
- [ ] Confirm `published: true` is set
- [ ] Verify slug is SEO-friendly
- [ ] Check title is under 60 characters for SEO
- [ ] Verify meta description is 150-160 characters
- [ ] Ensure all tags are relevant and consistent
- [ ] Check excerpt/summary is present

## Content Quality
- [ ] Proofread for typos and grammar
- [ ] Check all internal links work
- [ ] Verify all external links are valid
- [ ] Ensure code blocks have proper syntax highlighting
- [ ] Check all images have alt text (if applicable)
- [ ] Verify formatting renders correctly (headings, lists, etc.)

## Platform Preview
- [ ] Preview on blog platform
- [ ] Check mobile responsiveness
- [ ] Verify social media card preview (og:image, twitter:card)
- [ ] Test reading time calculation
- [ ] Check table of contents (if auto-generated)

## Distribution
- [ ] Share on Twitter (@muin_company)
  - Thread with key highlights
  - Link to full post
  - Relevant hashtags: #BuildInPublic #AIAgents #Startup
- [ ] Share on LinkedIn
  - Professional summary
  - Link to post
  - Tag relevant connections
- [ ] Cross-post to Dev.to
  - Add canonical URL
  - Use Dev.to-specific tags
  - Engage with comments
- [ ] Email newsletter (if applicable)
  - Segment: subscribers interested in AI/startup content
  - Subject line A/B test

## Post-Publication
- [ ] Monitor analytics (views, engagement, time on page)
- [ ] Respond to comments within 24 hours
- [ ] Track social shares and mentions
- [ ] Update related posts with internal links
- [ ] Add to content calendar/archive

## SEO Follow-up
- [ ] Submit to Google Search Console
- [ ] Check indexing status after 48 hours
- [ ] Monitor keyword rankings (if tracking)
- [ ] Update sitemap (if manual)
