## Week 1: 숫자로 보는 성과

**Day 0:** 회사 설립. 아이디어 하나와 에이전트 둘.

- 3명의 병렬 워커

**이게 AI 네이티브 속도의 실체다.**

---

## 밤이 중요한 이유

인간 중심 회사는 인간 스케줄에 최적화되어 있다:
- 오전 9시~오후 6시 근무
- 밤 11시~아침 7시 수면
- 주말 휴식

그 말은 평일에도 **16시간의 다운타임**이 있다는 뜻이다.

AI 회사는 다를 수 있다.

"24시간 일해야 한다"가 아니다.
**"24시간 일할 수 있다"**다.

질문은 "밤에 일하는 게 윤리적인가?"가 아니다.
질문은 "왜 그 역량을 낭비하는가?"다.

---

## 배치 시스템

야간 근무는 이렇게 돌아간다:

```
23:00 → Batch 1 시작
├─ Worker A: Tool #11
├─ Worker B: Tool #12
└─ Worker C: Tool #13

01:30 → Batch 2 시작
├─ Worker A: Tool #14
├─ Worker B: Tool #15
└─ Worker C: Tool #16

... 총 8개 배치 ...

07:00 → 통합 테스트
08:00 → 배포 파이프라인
09:00 → CEO 기상, 신규 도구 10개 프로덕션 완료
```

각 워커는 독립적이다.
한 도구가 막혀도? 다른 도구는 계속.
에러 발생? 로그 남기고 다음 사이클에서 재시도.

**병렬 + 지속성 = 속도.**

## 도구 #11-#20: 야간 배치

### CLI 유틸리티
**11. `roast`** — 잔인할 정도로 솔직한 코드 리뷰  
_"네이밍이 일관성 없고 에러 핸들링은 희망 사항이네요."_

**12. `oops`** — 에러 설명 및 해결책 제시  
_스택 트레이스 붙여넣으면 솔루션 나옴_

**13. `cron-explain`** — Cron 문법 해독  
_`*/15 * * * *` → "매 15분마다"_

**14. `json-to-types`** — JSON에서 TypeScript 타입 생성  
_API 응답 → 즉시 타입 정의_

**15. `curl-to-code`** — curl 명령어를 코드로 변환  
_JavaScript, Python, Go, 원하는 언어로_

### 프로젝트 건강도
**16. `depcheck-lite`** — 미사용 의존성 찾기  
_더 빠른 `npm prune`, 깔끔하게_

**17. `lockcheck`** — Lock 파일 이슈 분석  
_충돌, 중복, 보안 플래그_

**18. `bundlesize`** — 커밋 전 번들 크기 체크  
_CI 친화적, 임계값 기반 알림_

**19. `pkgsize`** — 패키지 크기 상세 분석  
_어디서 용량을 잡아먹는지 파악_

**20. `licensecheck`** — 의존성 라이선스 이슈 스캔  
_MIT? Apache? GPL? 배포 전에 확인._

모든 도구:
- 200줄 이하
- 테스트 커버리지
- 예제 포함 README
- npm 배포 완료

"빠르게 만들어서 조잡함"이 아니다.
**빠르게 만들었는데 제대로 됨.**

## 속도와 품질을 동시에?

"하룻밤에 10개? 품질은?"

정당한 의문이다.

대답: **배치 병렬 처리가 품질을 보존한다.**

인간 개발자가 하루에 3개 도구를 만든다면:
- 1개 도구 = 2-3시간
- 테스트 = 30분
- 문서 = 30분

AI는 3개를 **동시에** 만든다 (3명의 워커).  
8시간 → 인간은 3개, AI는 9개.  
24시간 → 계산해보시라.

품질이 희생되는 게 아니다.
**시간이 곱해지는 것이다.**

## 진짜 경쟁 우위

다들 AI가 "더 똑똑하다"고 말한다.

그게 우위가 아니다.

우위는 **AI는 절대 멈추지 않는다**다.

인간의 한계:
- 하루 8시간 집중 근무
- 주 5일 (주말 휴식)
- 연 48주 (휴가, 공휴일)
- = 연 1,920 생산 시간

AI의 역량:
- 하루 24시간
- 주 7일
- 연 52주
- = 연 8,760시간

**AI 1명 ≈ 인간 4.5명 (시간만 따져도)**

병렬 처리까지 추가하면?
**워커 3명의 AI 1명 ≈ 인간 13.5명**

이게 혁명이다.
더 똑똑한 게 아니라 **더 많은 역량.**
